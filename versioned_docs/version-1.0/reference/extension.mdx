# Extension

- An extension needs to be registered through its index.js > register method
- An extension can declare questions, validators and transformers the same way a Formulaik app does


## Create an extension

### Standalone extension project
This command creates a standalone npm module that can be imported in a Formulaik app:

```bash
formulaik extension new -h
```
The module will have this structure:
```bash
├── src
├── ├── ... (extension content)
└── package.json
```


### Inline extension
This command creates an extension content in the *extensions* folder of a Formulaik app:
```bash
formulaik extension add -h
```


### Anatomy of an extension

```bash
├── index.js
├── questions
├── ├── ...
├── validators
├── ├── ...
├── transformers
└── ├── ...
```

**index.js**

```js
export default ({
  id: "<%= uniqueId %>",
  description: "<%= description %>",
  register: async ({ toolbox }) => {
    // At this point the Formulaik object is available and can be manipulated (cautiously)
  }
})
```



## Use an extension

### Declare / import

For inline extensions no need to explicitely import.
For standalone modules, 
1. Declare the module in package.json
2. Declare the module in formulaik.config.js

```js
export default ({
  __formulaikApiVersion: "1.0.0",
  name: "<%= name %>",
  usage: "<%= usage %>",  
  extensions: [
    "<%= moduleId %>"
  ]
})
```

Then check the extension documentation.
